### 1. 渲染循环 (Render Loop)
在 App.render() 中每帧触发：

1. 清屏 ：清除颜色缓冲区和深度缓冲区。
2. 更新相机 ：获取当前的 View（视图）和 Projection（投影）矩阵。
3. 渲染场景 ( renderer.render )：
   - 绘制背景网格（Grid）。
   - 遍历场景中的所有零件并绘制。
4. 渲染高亮 ( render_selected_outline )：如果选中了零件，在其上方绘制橙色线框。
5. 渲染 UI ( imgui.render )：绘制文件浏览器、属性面板等 2D 界面。
6. 交换缓冲区 ：显示当前帧。
### 2. 场景渲染 (Renderer.render)
核心是 Shader（着色器） 的使用：

- Shader Program ：编译了 Vertex Shader (处理顶点位置、法线) 和 Fragment Shader (处理光照、颜色)。
- Uniforms ：向 Shader 传递 View、Projection 矩阵和光源位置。
- 绘制流程 ：
  1. 启用深度测试 ( GL_DEPTH_TEST )。
  2. 禁用背面剔除 ( GL_CULL_FACE )：因为 LDraw 模型常有非闭合几何体，开启会导致部分面透明。
  3. 遍历场景对象 ( scene.objects )：
     - 传递 Model 矩阵（对象的世界坐标变换）。
     - 绑定 VAO（顶点数组对象）。
     - 调用 glDrawArrays 绘制三角形。
### 3. 数据加载与上传 (upload_part)
- 懒加载 ：零件数据首次渲染时才上传到 GPU。
- VBO/VAO ：将 CPU 端解析好的顶点数据（位置、法线、颜色）打包存入 VBO (Vertex Buffer Object)，并通过 VAO 管理属性布局。
- 缓存 ：使用 self.vaos 字典缓存已上传的零件，避免重复上传。
### 4. 特殊效果
- 高亮轮廓 ：
  - 使用独立的 Shader ( outline_prog )。
  - 通过 glPolygonMode(..., GL_LINE) 绘制线框模式。
  - 使用 glPolygonOffset 将线框稍微拉向相机，防止与实体模型发生 Z-Fighting（闪烁）。
- 网格 (Grid) ：动态生成地面网格线，帮助定位。
